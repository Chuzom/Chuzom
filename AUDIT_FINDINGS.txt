BUG FOUND & FIXED
================================================================================

BUG #1: JSONL Query Crashes on Corrupt Data
Status: ✅ FIXED
Severity: HIGH

Location: src/chuzom/lineage/lineage_store.py:143-145
Issue: query_jsonl() threw JSONDecodeError on corrupt lines instead of
       skipping them gracefully
Impact: Would crash session-end hook if JSONL became corrupted
Fix: Added try/except to skip corrupt lines with graceful error handling

Before:
    for line in f:
        records.append(json.loads(line))  # ❌ CRASHES

After:
    for line in f:
        try:
            records.append(json.loads(line))
        except json.JSONDecodeError:
            pass  # ✅ SKIP

Test Added: test_corrupt_jsonl_line ✅ PASSED
